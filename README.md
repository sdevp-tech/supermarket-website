# سوبر ماركت الحارة — النظام المتكامل

نسخة مبسطة لنظام متجر إلكتروني باللغة العربية (اتجاه النص من اليمين إلى اليسار). هذا المشروع يحتوي على واجهة أمامية خفيفة وواجهة خلفية بسيطة جاهزة لإدارة المنتجات والطلبات والمستخدمين مع دعم تحميل الصور والنسخ الاحتياطي.

## المزايا
- واجهة عربية (RTL) جاهزة للعرض.
- واجهة خلفية مبنية بـ Node.js توفر REST API لإدارة: المستخدمين، المنتجات، العربة، الطلبات، الإشعارات، والتقارير.
- دعم رفع صور المنتجات وحفظها في مجلد `backend/uploads`.
- قاعدة بيانات محلية بسيطة باستخدام SQLite (ملف قاعدة البيانات داخل `backend/database/`).
- وظائف نسخ احتياطي وتصدير/استيراد البيانات بصيغة JSON.

## المتطلبات
- Node.js (الإصدار 14 أو أحدث موصى به)
- npm

## تشغيل المشروع محلياً (أقصر مسار)
1. استنساخ المستودع:
```bash
git clone https://github.com/sdevp-tech/supermarket-website.git
cd supermarket-website
```
2. تثبيت التبعيات:
```bash
npm install
```
3. تشغيل الخادم:
```bash
node backend/server.js
```
4. افتح المتصفح على:
```
http://localhost:3000
```

> ملاحظة: الخادم يقدّم الواجهة الأمامية من مجلد `frontend/` ويستخدم منفذ افتراضي 3000 (يمكن تغييره في الكود). ملف قاعدة البيانات الافتراضي: `./backend/database/database.db`.

## بيانات دخول تجريبية
- المدير: admin@example.com / admin123
- الموظف: employee@example.com / employee123
- العميل: customer@example.com / customer123

## بنية المشروع (مُختصَر)
```
README.md
package.json
backend/
  server.js          # خادم Node.js + نقاط النهاية (API)
  database/          # قاعدة بيانات SQLite (ملف .db يتم إنشاؤه تلقائياً)
  uploads/           # ملفات الصور المرفوعة
frontend/
  index.html
  js/app.js          # منطق واجهة المستخدم
  css/style.css
node_modules/        # تبعيات npm (ينصح بإضافته إلى .gitignore)
```

## إعدادات مُستحسنة ومواضيع تحتاج انتباه
- أضف ملف `.env` أو استخدم متغيرات البيئة بدل ثوابت داخل الشفرة (مثلاً: JWT_SECRET، PORT).
- أخرج `node_modules/` من المستودع وأضفه إلى `.gitignore` ثم ادفع التغييرات لتقليل حجم المستودع.
- تأكد من حماية ملف `backend/database/database.db` عند النشر وحفظ النسخ الاحتياطية في مكان آمن.
- مراجعة حدود رفع الملفات (الحد الحالي: 10MB) وسياسات أنواع الملفات المسموح بها.

## تطوير
- أنصح بفصل مسارات الخادم في مجلد `backend/routes/`، وفصل منطق الأعمال في `backend/controllers/` لتسهيل صيانة الكود.
- لإضافة اختبار سريع يمكن إضافة سكربت `test` في `package.json` وتشغيل إطار اختبار (مثل Mocha أو Jest).

## اقتراحات للنشر
- استخدم خدمة استضافة Node.js (مثل DigitalOcean, Heroku, Render) مع تشغيل الخادم خلف reverse proxy (nginx) وتهيئة HTTPS.
- استخدم متغيرات البيئة لإعداد قاعدة البيانات ومسارات التخزين والـ JWT secret.
