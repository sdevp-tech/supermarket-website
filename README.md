# سوبر ماركت الحارة — النظام المتكامل

نسخة مبسطة لنظام متجر إلكتروني باللغة العربية (اتجاه النص من اليمين إلى اليسار). يحتوي على واجهة أمامية خفيفة وواجهة خلفية بـ Node.js جاهزة لإدارة المنتجات والطلبات والمستخدمين مع دعم تحميل الصور والنسخ الاحتياطي.

## المزايا
- واجهة عربية (RTL) جاهزة للعرض.
- واجهة خلفية مبنية بـ Node.js + Express تقدّم REST API لإدارة: المستخدمين، المنتجات، العربة، الطلبات، الإشعارات، والتقارير.
- رفع آمن لصور المنتجات (JPEG, PNG, GIF, WEBP, BMP) مع فلترة حسب الامتداد ونوع MIME.
- قاعدة بيانات محلية SQLite (ملف قاعدة البيانات داخل `backend/database/`).
- نسخ احتياطي وتصدير/استيراد البيانات بصيغة JSON.
- إشعارات تلقائية للمخزون المنخفض (بدون تكرار).
- فصل صلاحيات ثلاثية: مدير / موظف / عميل، إضافة إلى دعم الطلبات كزائر.

## المتطلبات
- Node.js (الإصدار 14 أو أحدث موصى به)
- npm

## تشغيل المشروع محلياً

> ⚠️ **الخطوة 2 إلزامية الآن** — الخادم يرفض البدء إذا لم يكن `JWT_SECRET` موجوداً في ملف `.env`.

1. استنساخ المستودع:
```bash
git clone https://github.com/sdevp-tech/supermarket-website.git
cd supermarket-website
```

2. إنشاء ملف البيئة وسر JWT:
```bash
cp .env.example .env
# ولّد سراً قوياً (32 حرفاً على الأقل) وضعه في JWT_SECRET داخل .env
node -e "console.log(require('crypto').randomBytes(48).toString('base64'))"
```
أو يدوياً افتح `.env` واستبدل قيمة `JWT_SECRET` بسلسلة عشوائية طويلة. **لا ترفع ملف `.env` إلى Git** — هو مُضاف إلى `.gitignore`.

3. تثبيت التبعيات:
```bash
npm install
```

4. تشغيل الخادم:
```bash
npm start
# أو للتطوير مع إعادة التحميل التلقائي:
npm run dev
```

5. افتح المتصفح على:
```
http://localhost:3000
```

## إعدادات البيئة (`.env`)

| المتغير | الوصف | الافتراضي |
|---|---|---|
| `PORT` | منفذ تشغيل الخادم | `3000` |
| `JWT_SECRET` | سر توقيع رموز JWT — **إلزامي، 32 حرفاً على الأقل** | — |
| `MAX_UPLOAD_MB` | الحد الأقصى لحجم الصور المرفوعة بالـ MB | `10` |

> لا تلتزم بملف `.env` في Git. استخدم `.env.example` كقالب للمتغيرات المطلوبة.

## ملاحظات أمنية

تم تطبيق التحصينات التالية في الخادم:

- **JWT_SECRET** يُقرأ من `.env` ولا توجد قيمة افتراضية في الكود — الخادم يفشل صريحاً إذا غاب السر أو كان قصيراً.
- **رفع الملفات**: قائمة بيضاء لأنواع MIME والامتدادات المسموح بها؛ اسم الملف يُعاد بناءه من MIME لمنع تزوير الامتدادات.
- **تحديث المستخدمين** (`PUT /api/users/:id`): قائمة بيضاء للحقول القابلة للتعديل، تحقق من صحة البريد الإلكتروني ومن الأدوار المسموح بها، ومنع حقن `id` أو `is_active` في جسم الطلب.
- **طلبات الزوار**: السعر والاسم يُجلبان من قاعدة البيانات على الخادم — لا يُسمح للعميل بتحديد السعر.
- **كود طلب الزائر**: يُولَّد باستخدام UUID4 (إنتروبيا عالية) بدلاً من `Date.now() + Math.random()` لمنع التخمين.
- **إشعارات المخزون المنخفض**: لا تتكرر لنفس المنتج خلال 24 ساعة.
- **الملفات الحساسة** (`database.db`, `uploads/products/*`, `.env`) مُستبعدة من Git عبر `.gitignore`.

## تطوير

- لفصل الكود مع نمو المشروع، أنصح بنقل المسارات إلى `backend/routes/` ومنطق الأعمال إلى `backend/controllers/`.
- لإضافة اختبارات، أضف سكربت `test` في `package.json` (مثل Mocha أو Jest).

## اقتراحات للنشر

- استخدم خدمة استضافة Node.js (مثل DigitalOcean, Render, Railway) مع reverse proxy (nginx) و HTTPS.
- اضبط متغيرات البيئة في منصة الاستضافة — لا تنسخ `.env` المحلي.
- فعّل حماية إضافية: `helmet`، rate limiting، HTTPS فقط.
- احفظ نسخ احتياطية دورية من `backend/database/database.db` في موقع آمن منفصل.
- راجع دورياً سجل النشاط في جدول `activity_logs` لرصد أي سلوك غير معتاد.
